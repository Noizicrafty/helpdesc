# Just Enough Mails (JEM)

JEM - локальный инструмент для чтения писем из IMAP-ящика, очистки и нормализации текста, автоматической категоризации, выявления рекламно-спамовых признаков, генерации черновика ответа и выгрузки результата в CSV.

Проект ориентирован на персональный/командный разбор входящих писем за выбранный период, чтобы быстро получить «карту» переписки по темам и подготовить первичные ответы.

---

## Сквозная логика решения

Ниже представлен фактический runtime-пайплайн, который реализован в коде

1. **Загрузка конфигурации** из `runtime/*.json` (или создание файлов по умолчанию при первом запуске).
2. **Подключение к IMAP** с указанными email/паролем или app-password, host/port и SSL-флагом.
3. **Поиск писем по диапазону дат** через IMAP `SINCE/BEFORE`.
4. **Парсинг каждого письма**:
   - декодирование заголовков;
   - извлечение `From`, `Subject`, `Date`;
   - сбор текстового тела (`text/plain`, fallback на `text/html`, игнор вложений).
5. **Нормализация текста**:
   - удаление HTML/скриптов/стилей;
   - чистка URL/email/лишних символов;
   - оценка рекламно-спамового скора.
6. **Категоризация**:
   - по ключевым словам;
   - при включенном heatmap-модуле дополнительно - TF-IDF + cosine similarity;
   - при слабом совпадении предлагается новая категория.
7. **Генерация предложенного ответа**:
   - отдельный шаблон для подозрительного спама;
   - отдельные шаблоны для «проблем/ошибок», «продаж/счета»;
   - общий fallback-шаблон.
8. **Сохранение результата в память** `processed_emails` для просмотра в UI/CLI и дальнейшего экспорта.
9. **Экспорт CSV** по категориям-секциям в один файл `exports/mail_export_YYYYMMDD_HHMMSS.csv`.

## Зависимости проекта
- `beautifulsoup4>=4.12.0` - очистка HTML;
- `matplotlib>=3.8.0` - инфраструктура для тепловых карт;
- `scikit-learn>=1.4.0` - TF-IDF и cosine similarity.

## Архитектура модулей

### `maildesk/app_service.py`
Центральный orchestration-слой `MailProcessingService`:
- грузит/обновляет конфиг;
- поднимает IMAP-клиент;
- запускает нормализацию, классификацию, генерацию ответов;
- хранит текущую коллекцию `processed_emails`;
- отдает grouped-вид для UI;
- запускает экспорт.

### `maildesk/config_manager.py`
Управление конфигом в `runtime/`:
- `auth.json` - учетные данные + IMAP параметры;
- `settings.json` - флаги модулей/тема/шрифт/папка экспорта;
- `categories.json` - список категорий (имя, keywords, description).

Если файлов нет - создаются автоматически, вместе с дефолтными категориями.

### `maildesk/mail_client.py`
Низкоуровневый IMAP-клиент:
- соединение по SSL/без SSL;
- login + `INBOX`;
- ошибки аутентификации преобразуются в понятные сообщения в т.ч. подсказка про app-password;
- `fetch_emails_by_date_range` собирает письма и приводит к `EmailMessageData`.

### `maildesk/normalizer.py`
Препроцессинг текста:
- HTML -> text через BeautifulSoup;
- удаление ссылок/email/мусора;
- эвристический spam/ad scoring по regex-паттернам.

### `maildesk/classifier.py`
Категоризация письма:
- keyword score по каждой категории;
- при `enable_heatmap=True`: смешивание `semantic_score*0.7 + keyword_score*0.3`;
- порог слабого совпадения: `best_score < 0.22` -> предложение новой категории;
- извлечение top terms/values для объяснения.

> Важно: heatmap-данные используются в анализе и подготовке новых тем. Чувствительность может быть настроенка в коде решения.

### `maildesk/reply_suggester.py`
Генератор шаблонных ответов:
- branch для спама;
- branch для обращений про ошибки/инциденты;
- branch для коммерческих запросов;
- универсальный шаблон по умолчанию.

### `maildesk/exporter.py`
CSV-экспорт:
- группирует письма по категориям;
- пишет служебную строку-секцию `=== ТЕМА: ... ===`;
- добавляет колонки отправителя, темы, текста, даты, confidence;
- опционально добавляет `suggested_reply`.

### `maildesk/ui.py`
Tkinter desktop-интерфейс:
- ввод периода;
- запуск обработки;
- дерево «категория -> письма»;
- вкладки «Сводка / Письмо / Ответ»;
- окно настроек (auth, модули, темы, шрифт, категории);
- выбор папки и экспорт.

### `maildesk/cli.py`
Текстовое меню:
- настройка auth;
- настройка категорий;
- включение/выключение модулей;
- обработка диапазона дат;
- экспорт результата.

### `maildesk/models.py`
Dataclass-модели для всего пайплайна:
- `EmailMessageData`, `AssignmentResult`, `ProcessedEmail`, `AppSettings`, `CategoryConfig`.

## Конфиги и данные рантайма

Проект использует три JSON-файла для хранения служебных данных:

### `runtime/auth.json`
```json
{
  "email": "",
  "password": "",
  "imap_host": "imap.gmail.com",
  "imap_port": 993,
  "use_ssl": true
}
```

### `runtime/settings.json`
Содержит UI/модульные настройки и технические параметры (тема, шрифт, директория экспорта, флаги модулей).

### `runtime/categories.json`
Список стандартных категорий:
```json
[
  {
    "name": "Поддержка",
    "keywords": ["ошибка", "проблема", "не работает", "help", "support"],
    "description": ""
  }
]
```

`settings.json` и `categories.json` синхронизируются при сохранении настроек.

## Режимы работы

## CLI-режим
Пользователь в консоли:
- задает auth и IMAP параметры;
- настраивает категории и модули;
- запускает обработку диапазона;
- видит результаты + предлагаемые ответы;
- экспортирует CSV.

## UI-режим
Пользователь в окне:
- выбирает период;
- запускает обработку;
- кликает по письму в дереве;
- читает тело/сводку/ответ;
- экспортирует CSV в выбранную папку;
- меняет настройки и категории через отдельное окно.

## Формат категоризации

1. Для каждой категории считается доля совпавших ключевых слов (`keyword_score`).
2. Если heatmap-модуль выключен: берется лучший `keyword_score`.
3. Если heatmap-модуль включен:
   - строится корпус: документы категорий + текст письма;
   - считается TF-IDF;
   - считается cosine similarity письма к каждой категории (`semantic_score`);
   - итоговый `final_score = 0.7 * semantic_score + 0.3 * keyword_score`.
4. Если лучший итог `< 0.22` - письмо помечается как кандидат в новую категорию.

Дополнительно извлекаются top-термы для explainability (показываются в UI и в `AssignmentResult`).

## Рекомендуемый порядок первого запуска

1. Установить зависимости.
2. Заполнить `runtime/auth.json` (или через UI/CLI настройки).
3. Проверить/донастроить категории (`runtime/categories.json`).
4. Запустить обработку за короткий период (например, 1–3 дня).
5. Проверить, устраивает ли категоризация и текст автоответов.
6. Только после этого запускать на длинных интервалах и выгружать CSV.

## Безопасность

- Для Gmail/Google Mail используйте **app-password**, а не обычный пароль аккаунта.
- Не передавайте `runtime/auth.json`, так как данные хранятся в явном их виде.
- Если запускаете в Docker на shared-машине, ограничивайте доступ к volume `runtime/`.
